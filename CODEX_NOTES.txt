Codex notes for Villisika-Sepon juomapeli
========================================

Purpose of this file
--------------------
Read this first when continuing the project in a fresh Codex chat. It is a
memory refresh for the game's structure, current design direction, recent
changes, and a few things to avoid breaking. Do not put passwords, SMTP keys,
private tokens, personal account details, or other secrets in this file.

Current version
---------------
Game.version is currently 0.4.26-pikkukeikat.

README.txt should always mirror the actual Game.version in js/core.js whenever
the core game version is changed.

Core game idea
--------------
The high score is now total beers consumed, including beers consumed by helpers.
This is tracked as Game.state.oluet and synced to Supabase scoreboard columns
current_oluet / best_oluet through supabase-setup.sql.

The important economy split is:
- Eurot: money for beer, market items, gambling, Kela survival, and general
  short-term recovery. Talousindeksi affects euro prices/payments only.
  The live Talousindeksi is Supabase-backed and shared by all players.
- Ryypyt: progression currency for helpers, permanent upgrades, and random
  events. Ryypyt are no longer the scoreboard score and are not inflation-linked.
- Juodut oluet: the actual high-score number shown on the scoreboard.

Important project rules
-----------------------
- Keep changes scoped and avoid breaking Supabase auth/cloud saves.
- If Game.version changes, update index.html footer and README.txt too.
- After each meaningful patch, update CODEX_NOTES.txt when the change affects
  game direction, core systems, version history, Supabase behavior, or future
  handoff context.
- Do not commit private service credentials. Supabase/Brevo setup should stay
  in dashboards, not in repository notes.
- Supabase SQL changes usually require running supabase-setup.sql manually in
  the Supabase SQL Editor after pushing files.
- The project is a static browser game; opening index.html should work.

Main file map
-------------
- index.html: page shell, navigation, script loading, visible footer version.
- assets/seppo-logo.jpeg: sidebar logo shown at the top-left of the app.
- css/styles.css: shared page layout and visual styling.
- js/core.js: Game object, version, base state, module registration/update loop.
- js/ui.js: side navigation, guide overlay, top site switching, event ticker,
  optional event alarm sound, and quick stats.
- js/supabase-config.js: public Supabase client config.
- js/modules/player.js: drinking beer, producing ryypyt and empty cans.
- js/modules/beer-shop.js: beer prices, pack buying, price graphs.
- js/modules/economy.js: Supabase-backed shared euro-only economy/inflation
  index, with local fallback when the cloud is unavailable.
- js/modules/supermarket.js: food/drinks/cigarettes and hangover/stress relief.
- js/modules/returns.js: can return minigames and euro income.
- js/modules/jobs.js: rotating Pikkukeikat job board, mixed mission minigames,
  and economy-scaled euro income.
- js/modules/kela.js: benefit applications, delays, paperwork/calls.
- js/modules/stress.js: stress state and relief actions.
- js/modules/collapse.js: Romahdus and game-over flow.
- js/modules/helpers.js: Lauri, Jarski, Tuomas helper drinking/ryyppy output.
- js/modules/upgrades.js: permanent upgrades bought with ryypyt.
- js/modules/gambling.js: euro-based gambling minigames, including timed
  Karaokeilta availability for Karaoketuomio.
- js/modules/account.js: Supabase login/signup/password reset/cloud save UI.
- js/modules/save.js: save sanitation, always-on browser autosave, paced cloud
  autosave, and save/load state packaging.
- supabase-setup.sql: database tables, RLS, triggers, scoreboard view.

Recent version notes
--------------------
0.4.26-pikkukeikat
- Reworked Pikkukeikat from a five-button arrow-only list into a rotating
  three-job board. The board refreshes every 3 in-game days, and completed/
  failed jobs are replaced with a new job after a cooldown.
- Added four mission types:
  sequence arrow routines, typing forms, click-target jobs, and rotation/
  alignment jobs.
- Job rewards stay modest, roughly 1.40-7.30 euros before Talousindeksi
  scaling, so Pikkukeikat remain useful but should not overtake Kela, returns,
  or high-risk gambling.
- SaveLoad now stores jobsNextBoardRefreshDay and normalizes restored jobs so
  old saved job boards continue working in the new system.
- This is local gameplay/UI/save packaging only; no Supabase SQL/schema change
  is needed.

- Karaoke payouts were reduced to 1.5x, 2.1x, and 3x to keep it useful without

  change is needed.

0.4.23-prestige-sqlfix
- Fixed claim_prestige(run_oluet) in supabase-setup.sql so the profile table is
  aliased when updating lifetime_oluet. Without this, Supabase could fail
  prestige with "column reference lifetime_oluet is ambiguous" because the RPC
  also returns an output column named lifetime_oluet.
- Run the updated supabase-setup.sql again in Supabase to replace the function.

0.4.22-prestige
- Added an account-based Prestige system for logged-in players with a warning
  popup before reset. Prestige levels 1-10 show as:
  1-3 bronze stars, 1-3 silver stars, 1-3 gold stars, and level 10 platinum.
- Prestige requirements by next level:
  1000, 2000, 3500, 6000, 10000, 16000, 25000, 40000, 65000, 100000 juodut oluet.
- Client flow: Account.claimPrestige pauses autosave, calls the
  claim_prestige(run_oluet) Supabase RPC, clears browser autosave, then reloads
  into a fresh run.
- Supabase flow: supabase-setup.sql adds profiles.prestige_level,
  lifetime_oluet, last_prestiged_at, prestige_required_oluet(), and
  claim_prestige(). The RPC increments prestige, resets profile current/best
  run scores, and deletes the old main cloud save so it cannot be reloaded.
- Scoreboard now selects prestige_level and lifetime_oluet and keeps prestiged
  players visible even when their current prestige-run score is 0.
- This version requires running the updated supabase-setup.sql in Supabase.

Important current behavior
--------------------------
Romahdus:
- Triggered by severe failure state: stress 100, hangover 100, euros below 1,
  no full cans, and not already collapsed that same day.
- First and second Romahdus halve euros and ryypyt, give a small rescue stock,
  push the day forward, and reset some pressure.
- Third Romahdus sets gameOver true. It does not automatically cloud-save.

Permanent upgrades:
- Upgrades are currently saved as part of the save data.
- A fresh restart/new save loses them unless a cloud/local save containing them
  is loaded.

Scoreboard:
- The score that matters is juodut oluet, including helper drinking.
- The game has fallback code for older Supabase schemas that only have ryypyt,
  but the intended live schema is the olut-score version.

Gambling:
- Uses euros for bets and payouts.
- Should not directly create or remove ryypyt.
- Stress/hangover consequences are still allowed.

Supabase/auth notes without secrets
-----------------------------------
- Auth, cloud save, profiles, and scoreboard are Supabase-backed.
- Global economy is also Supabase-backed through get_global_economy(); players
  should not get direct write access to public.global_economy.
- Password reset exists in account.js.
- Custom SMTP was configured outside the codebase and should not require game
  code changes unless redirect behavior changes.
- Deleting an auth user should cascade profile/save rows if the current SQL
  setup is in place.
